# خطوات النشر — التطبيق يعمل من أي جهاز وأي شبكة

## الخطوة 1: إنشاء مخزن على GitHub

1. ادخل إلى [github.com](https://github.com) وسجّل الدخول.
2. اضغط **New repository** (أو الزر الأخضر **+**).
3. اسم المخزن: `karas-magdy` (أو أي اسم تحب).
4. اختر **Public**.
5. **لا** تضف README أو .gitignore (موجودة فعلاً).
6. اضغط **Create repository**.

---

## الخطوة 2: رفع المشروع

في PowerShell أو Terminal من داخل مجلد المشروع نفّذ:

```powershell
cd D:\programs\whatsapp-clone-backend

git remote add origin https://github.com/YOUR_USERNAME/karas-magdy.git

git branch -M main

git push -u origin main
```

استبدل `YOUR_USERNAME` باسم حسابك على GitHub.

---

## الخطوة 3: النشر على Render.com

1. ادخل إلى [render.com](https://render.com) وسجّل الدخول (بإمكانك استخدام حساب GitHub).
2. اضغط **New +** ثم اختر **Web Service**.
3. اختر **Connect a repository** واربط مخزن `karas-magdy`.
4. Render سيقرأ `render.yaml` تلقائياً.
5. في **Environment Variables** أضف:

| المتغير      | القيمة |
|-------------|--------|
| `MONGO_URI` | انسخ من ملف `.env` المحلي (السطر الكامل الذي يبدأ بـ `mongodb+srv://`) |
| `JWT_SECRET`| أي نص عشوائي طويل (مثال: `mySecretKey123!@#`) |
| `ADMIN_PHONES` | `01229084204,1229084204` |

6. اضغط **Create Web Service**.
7. انتظر حتى ينتهي البناء والنشر (~3–5 دقائق).

---

## الخطوة 4: تعيين BASE_URL بعد النشر

1. بعد نجاح النشر، انسخ رابط التطبيق (مثال: `https://karas-magdy-xxxx.onrender.com`).
2. في Render → **Environment** → أضف متغيراً جديداً:
   - المتغير: `BASE_URL`
   - القيمة: `https://karas-magdy-xxxx.onrender.com` (الرابط الخاص بك)
3. من القائمة العلوية اختر **Manual Deploy** → **Redeploy** لإعادة التشغيل بالإعدادات الجديدة.

---

## النتيجة

التطبيق سيعمل على رابط عام مثل:
`https://karas-magdy-xxxx.onrender.com`

يمكن الوصول له من:
- أي موبايل
- أي شبكة (واي فاي، بيانات الجوال)
- أي مكان في العالم
